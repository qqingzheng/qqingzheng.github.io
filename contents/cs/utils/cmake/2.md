# 编译多个文件

## 编译同目录下多个文件

### 文件结构

```cpp
//Example.cpp
#include <iostream>
#include "Add.h"
using namespace std;
int main(){
	cout << add(10,2);
	return 0;
}
```

```cpp
//Add.h
int add(int,int);
```

```cpp
//Add.cpp
#include "Add.h"
int add(int a,int b){
    return a+b;
}
```

### CMakeLists.txt

```
cmake_minimum_required(VERSION 3.10)

project(demo)

add_executable(example Example.cpp Add.cpp)
```

如果文件太多，可以用`aux_source_directory(<目录> <变量名>)`，将目录下所有源文件添加入变量

```
cmake_minimum_required(VERSION 3.10)

project(demo)

aux_source_directory(. SRC_LIST)

add_executable(example ${SRC_LIST})
```

但是如果目录下有我们不需要的文件，则使用`set(<变量名> <变量内容>)`命令去定义新变量：、

```
cmake_minimum_required(VERSION 3.10)

project(demo)

set(SRC_LIST
	./Example.cpp
	./Add.cpp)
	
add_executable(example ${SRC_LIST})
```

## 编译不同目录下多个文件

### 文件结构

├── Add
│   ├── Add.cpp
│   └── Add.h
├── CMakeLists.txt
├── Example.cpp

### CMakeLists.txt

```
cmake_minimum_required(VERSION 3.10)

project(demo)

set(SRC_LIST
	Add/Add.cpp
	./Example.cpp)
	
include_directions(Add)

add_executable(example ${SRC_LIST})
```

或者

```
cmake_minimum_required(VERSION 3.10)

project(demo)

aux_source_directory(Add SRC_LIST)

include_directories(Add)

add_executable(example Example.cpp ${SRC_LIST})
```

这里我们要注意`include_directories(<目录>...)`，这个语句可以将目录加入头文件搜索，编译器就会在这个目录搜索头文件。

## 正规的编译结构

### 文件结构

.
├── bin
├── build
├── **CMakeLists.txt**
├── include
│   ├── Add.h
│   └── Minus.h
└── src
    ├── Add.cpp
    ├── **CMakeLists.txt**
    ├── Example.cpp
    └── Minus.cpp

### CMakeLists.txt

```
#项目根目录下的CMakeLists.txt
cmake_minimum_required(VERSION 3.10)

project(demo)

add_subdirectory(src)
```

注意`add_subdirectory(<目录>...)`，这个语句可以让CMakeLists.txt到该目录下执行另一个CMakeList.txt

```
#项目根目录下的src/CMakeLists.txt
aux_source_directory(. SRC_LIST)

include_directories(${PROJECT_SOURCE_DIR}/include)

add_executable(example ${SRC_LIST})

set (EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/bin)
```

注意`set(<变量> <数据>)`，可以将变量设置为某个数据。其中`EXECUTABLE_OUTPUT_PATH`为cmake的预定义变量，即可执行文件的输出目录。

上面代码也可以集中在一个CMakeLists.txt中：

```
cmake_minimum_required(VERSION 3.10)

project(demo)

aux_source_directory(src SRC_LIST)

include_directories(include)

add_executable(example ${SRC_LIST})

set (EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/bin)
```
