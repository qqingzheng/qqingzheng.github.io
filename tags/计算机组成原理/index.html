[计算机组成原理] 3-2ISA

ISA的组成

ISA（Instruction Set Architecture，指令体系架构）由寄存器集、寻址方式和指令格式组成。

寄存器

假如没有寄存器：假设一条指令ADD r0, r1, r2，这条指令操作码（ADD）为16位，地址为32位，则总长为112位。而计算机字通常为16，32或64位，因此指令过长不行。并且，因为没有寄存器，CPU发出的32位地址到指定的存储器还需要时间（中间还需要地址译码），所以访问时间会很长。

寄存器与其他存储单元除了访问的便捷性和响应时间的快速之外没有区别。并且可以使用很短的指令位就可以指定寄存器。如下图，某计算机操作码8位，地址位3位，额外的位用来当控制字段。

寄存器的大小通常等于计算机完成数据处理的最大位宽。例如16位计算机寄存器为16位，可以完成两个16位的加法。

如果对寄存器的部分内容进行操作，一些处理器可能会有以下的问题：

通用寄存器与特殊功能寄存器的比较：通用寄存器需要使用几个位去指定功能，而特殊功能寄存器与某些专用功能绑定在一起，无需在指令字中分配位。

寻址方式

三种类型：立即数寻址、直接寻址、间接寻址。

立即数寻址即直接将常数作为指令的一部分。

直接寻址则如ADD P, Q, R将存储单元Q的内容与存储单元R的内容直接相加后写入存储单元PP，则将操作数地址作为指令的一部分。

间接寻址也称为寄存器间接寻址，即寄存器地址（指令寄存器）为指令的一部分，这个寄存器中包含了操作数地址。间接寻址对于访问数组很有帮助，因为可以操作寄存器进行运算。

微处理器提供的指令格式一般有三种：存储器->寄存器，寄存器->存储器，寄存器->寄存器。

指令格式

RISC计算机的指令长度受到严格限制，即指令必须为一个字，如32位、64位等。CISC计算机允许指令长度拓展为几个字。因此如果设计者希望所有指令包含三个字段：操作、立即数、存储单元位置。如果立即数支持0-900，存储单元0-250000，那么需要占用28位指令位，这样就只有4个位可以用来代表操作，也只有16种操作。

操作码与指令

双地址计算机

CISC计算机采用双地址指令格式。它不能实现[P] <- [Q] + [R]，只能实现[Q] <- [P] + [Q]。所以ADD P,Q会破坏Q原本的内容。

单地址计算机

单地址计算机中有叫累加器的固定寄存器。可以先将操作数载入累加器，然后再用另外的数与累加器中的数相加，然后将累加器中的结果保存到其他地址。这样可以节省指令位，早期的8位计算机就采用的单地址的方式。

[计算机组成原理] 3-1存储程序计算机

计算机体系结构

计算机体系结构通常是指程序员视角中的计算机。程序员所看的计算机是抽象的，脱离了繁琐的底层物理实现等。对他们而言1+1这样的操作是通过指令完成的，而不是复杂的数字电路。计算机体系结构的这个抽象视图也被称为指令集体系结构。

计算机组成是计算机体系结构的具体实现（实例化）。也就是说，计算机体系结构是所有计算机的一个抽象结构，而计算机组成则是这个抽象结构的实例化。
