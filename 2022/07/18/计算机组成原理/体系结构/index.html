[计算机组成原理] 体系结构与组成

0.计算机体系结构

计算机体系结构通常是指程序员视角中的计算机。程序员所看的计算机是抽象的，脱离了繁琐的底层物理实现等。对他们而言1+1这样的操作是通过指令完成的，而不是复杂的数字电路。计算机体系结构的这个抽象视图也被称为指令集体系结构。

计算机组成是计算机体系结构的具体实现（实例化）。也就是说，计算机体系结构是所有计算机的一个抽象结构，而计算机组成则是这个抽象结构的实例化。

1.存储程序计算机

ARM这一类处理器采用了存储程序体系结构，将程序和数据放在同一个储存空间中。其按顺序从内存读指令、译码、执行。这样一台计算机带有寄存器、算术逻辑单元（ALU）、储存器以及连接各个部分的总线。

寄存器是CPU内部的存储单元，寄存器使用名字而不是地址来访问。指令中用来选择寄存器的字段一般为3~5位（因此寄存器比较少）。

寄存器往往有几个功能：

  1. 高速暂存：用于保存数据的指针。
  2. 特殊功能：循环计数器、状态寄存器、栈指针寄存器和CPU标识寄存器等。
  3. 程序计数器（PC）：记录下一条指令的地址。
  4. 不可见寄存器（不属于处理器体系结构）：存储器地址寄存器（MAR）正在访问的存储器地址；存储器数据寄存器（MBR）保存了要从存储器读出的数据或要写入的数据；指令寄存器（IR）：记录当前正在执行的指令

计算机指令的规定：为了简易通用，计算机指令规定在书上P80写有。

  1. LDR 寄存器目的, 存储源
  2. STR 寄存器源, 存储目的
  3. ADD/SUB/AND... 寄存器目的, 寄存器源1, 寄存器源2
  4. BPL 指令目的 如果前一个操作的结果大于等于0，那么就会跳转到新的指令地址（而不是PC的+4）。
  5. BEQ 指令目的 如果前一个操作结果为0，则跳转。
  6. B 指令目的 直接跳转。

扩展处理器：常量处理

立即数是一个在运算中可以直接使用的数，它与存储单元中的值不同。例如LDR r1,#200，将200这个常量直接写入r1寄存器中，而不是读地址为200的存储器内容。

立即数是直接从IR的操作数传输到ALU的，并不是由存储系统经过MBR传输。

ALU的信息将被写入条件码寄存器（CCR，Condition Code Register），它保存了用于测试的条件（零位、借位位、负位和溢出位）。CCR与CU连接。CU负责指令译码，并且生成控制信号执行指令。CU也会从CCR中选择所需的条件位进行测试。如果测试为false则会按照PC+4执行指令；如果为true则会跳转到指定的地址（分支目标地址BTA，Branch Target Address）。

CCR状态信息

计算机的ALU执行一个操作时（不同的处理器情况不同，有的需要程序员要求其更新CCR），处理器会记录下结果是否为0（Z，零位），是否为负（N，负位），是否产生进位位（C，进位位），是否算数溢出（V，溢出位）。
